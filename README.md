# Java 學習歷程與物件導向程式設計實作 (Java Learning Journey & OOP Implementation)

這是我從 **數據科學（淡江統計資科所）** 跨足軟體工程領域的 Java 自學紀錄。本專案不僅包含基礎語法練習，更著重於如何透過 **物件導向 (OOP)** 的設計思維，將複雜的商務邏輯（如金融、保險、資產管理）轉化為穩定且具擴充性的程式架構。

---

## 核心學習範疇

### 1. 物件導向四大特性 
我深入研究了 Java 的核心特性，並將其應用於模擬金融系統的開發：
* **封裝 (Encapsulation)**：使用 `private` 變數配合 `getter/setter` 確保資料安全性。
    * *實作：Ch06 & Ch13 透過保戶資料管理，練習存取權限控制與邏輯檢查。*
* **繼承 (Inheritance)**：透過父子類別結構實現代碼重用。
    * *實作：Ch07 建立基礎保單類別，讓不同險種繼承共同屬性，減少冗餘。*
* **多型 (Polymorphism)**：讓同一個指令在不同物件上展現不同行為，提升系統彈性。
    * *實作：Ch08 & Ch14 處理異質物件（如投資型保單、人壽保險）的統一計算邏輯。*
* **抽象 (Abstraction)**：利用抽象類別與介面定義系統規範，實現架構與實作分離。
    * *實作：Ch08 & Ch12 定義金融服務的標準規範（如驗證、執行流程）。*

### 2. 資料結構與集合框架 
* **動態管理**：熟練使用 `ArrayList` 管理變動的交易數據。
* **快速檢索**：在 `Ch11` 實作 `HashMap` 練習，模擬金融系統中透過「客戶編號」快速查詢「保戶資料」的高效能邏輯。
* **權責分離**：練習將邏輯類別與資料管理類別分離，模擬真實後端開發架構。

### 3. 例外處理 (Exception Handling)
* **系統穩定性**：在 `Ch10` 針對運算異常（如除以零）進行捕捉與處理，確保金融運算過程中程式不會意外崩潰。

身為數據科學所的學生，我習慣於嚴謹的邏輯推演。Java 的「強型別」與「物件導向」特性，能將這種嚴謹性落實在大型軟體開發中。

我追求的不只是跑出結果，更是構建出一個 **穩定**、**整潔** 且 **易於維護** 的系統架構。這份學習歷程是我將數據思維轉化為工程實力的最佳見證。
