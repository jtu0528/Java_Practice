# Java 學習歷程與物件導向程式設計實作 (Java Learning Journey & OOP Implementation)

這是我從數據分析（統計學背景）跨足軟體工程領域的 Java 自學紀錄。本專案不僅包含基礎語法練習，更著重於如何透過 **物件導向 (OOP)** 的設計思維，將複雜的商務邏輯（如金融、資產、帳務處理）轉化為穩定且具擴充性的程式架構。

## 核心學習範疇

### 1. 物件導向程式設計 (The Four Pillars of OOP)
我深入研究了 Java 的四大核心特性，並理解其在大型系統中的必要性：
* **封裝 (Encapsulation)**：使用 `private` 變數配合 `getter/setter` 方法，確保資料安全性，防止外部未授權的存取與修改。例如：在 `Ch06` 實作保戶年齡與金額的合法性檢查。
* **繼承 (Inheritance)**：透過父類別與子類別的結構，實現代碼重用。例如：在 `Ch07` 建立基礎保單類別，讓不同險種繼承其共同屬性。
* **多型 (Polymorphism)**：透過方法重寫（Overriding），讓同一個指令在不同物件上展現不同行為，提升系統彈性。例如：在 `Ch08` 處理股票與債券的統一估值。
* **抽象 (Abstraction)**：利用抽象類別 (`abstract class`) 與介面 (`interface`) 定義系統規範，實現「抓大放小」的架構設計。例如：在 `Ch08` 規範金融服務必須包含審核與執行流程。

### 2. 系統架構設計邏輯 (System Design Logic)
我練習將實務問題抽象化。以「金融運算系統」為例：
* 定義 `abstract class FinancialService` 作為所有業務的設計圖。
* 定義抽象方法 `processRequest()`，讓各子類別（如：貸款、開戶）根據自身業務邏輯進行具體實作。
* 這種設計確保了系統在新增功能時，不需要修改既有的核心代碼，符合軟體開發的「開閉原則 (Open-Closed Principle)」。

### 3. 資料結構與集合框架 (Java Collections Framework)
* **動態管理**：熟練使用 `ArrayList` 等集合工具管理變動的交易數據。
* **權責分離**：在 `Ch09` 中實作邏輯類別與管理類別分離，模擬真實後端開發架構。
* **統計結合**：結合統計背景，練習如何高效地處理與篩選大規模物件資料集。
