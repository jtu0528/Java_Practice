Java 學習歷程與物件導向程式設計實作
(Java Learning Journey & OOP Implementation)
這是我從數據分析（統計學背景）跨足軟體工程領域的 Java 自學紀錄。本專案不僅包含基礎語法練習，更著重於如何透過 物件導向 (OOP) 的設計思維，將複雜的商務邏輯（如金融、資產、帳務處理）轉化為穩定且具擴充性的程式架構。

🚀 為什麼選擇 Java？
身為統計研究所的學生，我習慣於嚴謹的邏輯推演。Java 的「強型別」與「物件導向」特性，能將這種嚴謹性落實在軟體開發中。我追求的不只是跑出結果，更是構建出一個穩定、整潔且易於維護的系統架構。

🛠 核心實作：10 章進階學習地圖
本儲存庫按照學習的時間序與邏輯層次，分為三大階段：

第一階段：📂 Ch01_OOP_Basics (物件導向與核心邏輯)
這部分展示了從基礎語法進化到 OOP 架構的過程。

Ch 01 | 基礎算術與薪資計算：掌握變數宣告與金融資料型別處理。

Ch 02 | 條件判斷與稅務級距：實作多重分支 if-else 邏輯。

Ch 03 | 迴圈控制與年度結算：運用累加器 (+=) 處理批次運算。

Ch 04 | 傳統陣列與保費管理：理解索引存取與固定長度資料處理。

Ch 05 | 方法宣告與邏輯分離：將業務規則封裝成 Method，實作邏輯重用。

Ch 06 | 封裝 (Encapsulation)：使用 private 與 getter/setter 保護敏感帳務資料。

Ch 07 | 繼承 (Inheritance)：實作父子類別，展示程式碼重用性。

Ch 08 | 抽象 (Abstraction) 與 多型：

利用抽象類別定義系統規範（如：金融服務申請流程）。

透過多型處理異質資產（如：股票與債券的統一估值）。

第二階段：📂 Ch02_Collections_ArrayList (動態資料管理)
Ch 09 | ArrayList 與 管理員架構：實作 CRUD 邏輯，展示動態交易紀錄的處理能力。

第三階段：📂 Ch03_Exception_Handling (系統穩健性)
Ch 10 | 例外處理與防禦性編程：透過 try-catch-finally 捕捉異常（如：除以 0、非法輸入），確保系統不因單點錯誤而潰退。

🧠 我的設計思維 (System Design Logic)
規範化 (Standardization)：
利用 abstract class 定義設計圖，強制子類別實作核心邏輯（如：身份驗證與請求處理），確保系統安全。

封裝性 (Security)：
所有金融變數皆設為 private，並在 Setter 中加入範圍檢查（如：年齡限制、金額不為負），從代碼層級確保資料正確性。

擴充性 (Scalability)：
運用多型（Polymorphism）設計「一個介面，多種實作」，確保系統在新增險種或服務時，不需要修改既有的核心代碼。
